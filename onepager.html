きっかけと、復活の裏側

きっかけは Amazon の研究者が報告した、セーフガードを回避する手法（jailbreak）でした ✅。Anthropic は「狭い範囲の jailbreak の発見が、数億人に展開された商用モデルの回収理由となるべきではない」と異議を表明しつつ ✅、復活にあわせて改良版の安全分類器を導入。公式発表では「報告された手法を 99%以上のケースでブロック」し、ブロック時は Opus 4.8 に振り替えられます ✅。

Fable 5 とは — Opus の上の「Mythosクラス」 ✅

Claude 5 ファミリー最初のモデルで、これまで最上位だった Opus のさらに上の階層に位置します。公式発表いわく「タスクが長く複雑なほど優位性が顕著」。

Fable 5 と Mythos 5 は同じ基盤モデルで、違いは安全対策の層だけ。一般提供されるのは Fable 5、Mythos 5 は承認済み組織のみです ✅。

まずやること① Skills・CLAUDE.md の一気修復 🔶

停止中の19日間で…
  • 相対日付（「明日やる」等）が期限切れ
  • 削除済み手順への参照が残骸化
  • ルールの重複・矛盾が蓄積
→
修復してから任せる
  • 高性能モデルは指示を忠実に読む
  • ノイズ入り設定はノイズまで実行される
  • モデル切り替え時が大掃除の好機
修復プレイブック「memory-dream」（conao_3 氏・X発 🔶）

① Mine（繰り返し出た指摘・確定方針だけ抽出）→ ② Consolidate（相対日付→絶対日付・矛盾は最新で上書き）→ ③ Dedup（上位のルールを下位から削除）→ ④ Prune(索引は200行以内) の4フェーズ。鉄則は「経緯・失敗談は書かず、今正しい知見だけ残す」。

まずやること② ハーネス構築 — 公式3記事から読み解く ✅

ハーネス＝モデルの周りに置く「手綱」一式（CLAUDE.md・スキル・フック・検証の仕組み）。エンジンが強くなるほど手綱側の設計が成果を左右します。教科書は2026年6月に公開された公式ブログ3本。

A

Steering Claude Code

6/18公開。指示を置く「7つの場所」と使い分け

B

Human-Agent Teams

6/24公開。エージェントを同僚として運用する教訓

C

Getting started with loops

6/30公開。4種類のループと「止め方」の設計

公式が挙げる「指示の置き場所を見直すサイン」 ✅
  • 「毎回Xなら必ずY」を CLAUDE.md に書いている→フックへ（判断に頼らず自動実行）
  • 「してはいけない」を指示で書いている→フック＋権限設定で強制する
  • 30行の手順を CLAUDE.md に書いている→スキルへ（呼び出し時だけ読む）
  • 個人の好みをプロジェクト設定に書いている→ユーザーレベルのファイルへ
ループは4種類。今回のハーネス構築は「①ターン型」で1回走らせるだけ ✅

① ターン型（普段の対話）／② ゴール型（/goal で完了条件を与え自走）／③ 時間型（/loop・/schedule）／④ プロアクティブ型（イベント起点で自律稼働）。構築したハーネスが、後々 ②〜④ の自律ループを安全に回す土台になります。
