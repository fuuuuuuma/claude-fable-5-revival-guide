なぜ止まったのか

きっかけ ✅

Amazon の研究者が、Fable 5 のセーフガードを回避してソフトウェアの欠陥を特定させる手法（jailbreak）を報告。米政府はこれを受けて輸出管理指令を発出した。

停止の範囲 ✅

対象は Fable 5 / Mythos 5 のみ。国籍をリアルタイム確認する手段がないため、Anthropic は全ユーザーへの提供を一時停止した。Opus 4.8 など他モデルは影響なし。

Anthropic は「狭い範囲の jailbreak の発見が、数億人に展開された
商用モデルの回収理由となるべきではない」と異議も表明 ✅

  • 相対日付（「明日やる」等）が期限切れ
  • 削除済み手順への参照が残骸化

  • 切り替えのタイミングが大掃除の好機

そのまま使える修復プロンプト 🔶

# CLAUDE.md / Skills / メモリの一気修復
> memory-dream を実行して、記憶階層全体を整理してください。
> ① 繰り返し出た指摘・確定方針だけ抽出（1回限りのメモは無視）
> ② 相対日付→絶対日付、矛盾は最新で上書き、死んだ参照は削除
> ③ 上位ファイルのルールを下位から削除。メタ説明は残さない
> ④ 索引は200行以内に。経緯・失敗談は書かず、今正しい知見のみ
> 出力はレビューできる形で、変更をコミットとして提示して

結果は自分でレビューしてから確定する（勝手に確定させない）

ハーネス構築プロンプト（要約版）

# リポジトリのハーネスを公式基準で診断→再設計（①ターン型で1回実行）
> このリポジトリの Claude Code 設定を診断してください。基準:
> ・CLAUDE.md は「常に必要な事実」だけ・200行未満
> ・「毎回Xなら必ずY」はフックへ／禁止事項はフック＋権限で強制
> ・30行超の手順はスキルへ／パス限定の規約は paths 付きルールへ
> ・検証手順（テスト・チェックリスト）をスキルとして書き出す
> まず「どの指示を・どこへ・なぜ動かすか」の一覧表を提示。
> 私が承認したら1項目ずつ移行してください。

全文は README に掲載。「診断→一覧表→承認→移行」の順で人間のレビューを挟むのがポイント
